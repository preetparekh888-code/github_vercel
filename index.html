The Two Tools

🐙

💻 Step 1 — You write code on your computer

Open your index.html in Notepad, VS Code, or any text editor. Make your changes — change a color, add a button, fix a typo. Save the file normally with Ctrl+S. That's it for your part here. Your file only exists on your computer at this point — nobody else can see it yet.

⬆️ Step 2 — Push your files to GitHub

Go to your GitHub repo in the browser → click Add file → Upload files → drag your updated index.html → click Commit changes. GitHub now stores your new version. This is called a "push" or "commit." GitHub keeps this version forever alongside all your previous ones.

👁 Step 3 — Vercel detects the change

The moment you commit to GitHub, Vercel receives an automatic notification called a "webhook." You don't do anything — Vercel is always watching your repository 24 hours a day, 7 days a week. Within 1-2 seconds of your commit, Vercel starts its deployment process automatically.

⚙️ Step 4 — Vercel builds and deploys

Vercel pulls your latest code from GitHub, processes it (for a plain HTML site this is instant — no build step needed), and distributes it across its global network of servers in cities around the world. This takes about 10-30 seconds. You can watch the live build log in your Vercel dashboard.

🌍 Step 5 — Your site is live worldwide

Your URL (like yoursite.vercel.app) now serves your latest code. Someone in India, USA, or Japan all get the same fast experience because Vercel serves from the nearest server to them. The old version is replaced instantly. Every future push repeats this cycle automatically.

Common Questions

Things people always ask

Do I need to pay for any of this? +
No. GitHub is free for public and private repos. Vercel's Hobby plan is completely free and includes unlimited deployments, HTTPS, and a vercel.app subdomain. You only pay if you need a custom domain (like yourname.com) or enterprise features.
What happens if I push bad code — does my site break? +
Vercel keeps your previous deployment running until the new one succeeds. If your build fails, the old version stays live and you get an email. You can also click "Rollback" in Vercel's dashboard to instantly go back to any previous version.
Can I use a custom domain like mysite.com? +
Yes! In Vercel → your project → Settings → Domains → add your domain. You'll need to buy the domain from somewhere like Namecheap or GoDaddy (usually $10-15/year), then point it to Vercel. Vercel handles the HTTPS certificate for free automatically.
Do I need to install anything on my computer? +
No! For a basic HTML/CSS/JS site, you can upload files directly through GitHub's website (drag and drop). No terminal, no Git installation needed. Just browser → GitHub → drag file → commit → Vercel deploys automatically.
What's the difference between GitHub and Vercel exactly? +
GitHub = storage (like a hard drive in the cloud for code). Vercel = publisher (turns that stored code into a live website). You need both because storing code and serving it to millions of users are two very different technical problems.
